▼ Technical details (for analyst review)

Account Fixation Complete

The victim's browser is now logged into the attacker's KAYAK account.

Open kayak.com/profile/payment — this is the attacker's account. Any credit card details entered here are stored in the attacker's account and can be retrieved by the attacker later.

What happened

The phishing link opened this page, which sent a cross-origin postMessage to kayak.com/login containing the attacker's Google OAuth token. KAYAK's login handler checks event.data.action === "googleLogin" but does not check event.origin. It processed the attacker's token and set the session to the attacker's account.

// Google OAuth — NO origin check: "googleLogin"===t.action ? (i&&i.close(), t.access_token ? n(t) : o(t)) : o(e) // Microsoft handler (same codebase) — HAS origin check: if (e.origin !== expectedOrigin) return;
